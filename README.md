# 🤖 Telegram Support Bot

Telegram-бот для автоматизации обработки заявок в службу технической поддержки. Бот собирает информацию о проблемах пользователей, верифицирует сотрудников и автоматически перенаправляет заявки в dedicated канал поддержки.


## ✨ Возможности

- **🔐 Верификация сотрудников** через email и код подтверждения
- **📋 Структурированный сбор заявок** с категоризацией проблем
- **📊 Автоматическая отправка заявок** в Telegram-канал поддержки
- **💾 Сохранение данных пользователей** в базе данных SQLite
- **📱 Удобный интерактивный интерфейс** с кнопками и меню
- **⏰ Таймауты и уведомления** о неактивности пользователя

## 🗂️ Категории проблем

Бот поддерживает 7 основных категорий проблем с подкатегориями:

1. **📧 Эл. почта** - проблемы с почтой
2. **💙 ПО 'Синтез'** - вопросы по корпоративному ПО
3. **🖥️ Проблема с ПК** - неисправности компьютеров
4. **☎️ Телефон** - проблемы с телефонией
5. **🖨️ Принтеры, МФУ** - неисправности печатающей техники
6. **🌐 Удаленный доступ** - настройка AnyDesk
7. **❓ Другое** - прочие вопросы

## 🛠️ Технологический стек

### Основные технологии
- **Python 3.8+** - основной язык программирования
- **python-telegram-bot 20.7** - работа с Telegram Bot API
- **SQLite3** - база данных пользователей
- **SMTP** - отправка email с кодами подтверждения
- **Docker** - контейнеризация

### Вспомогательные библиотеки
- **python-dotenv 1.0.0** - управление переменными окружения
- **requests 2.31.0** - отправка HTTP запросов
- **pandas 2.1.3** - работа с Excel файлами
- **openpyxl 3.1.2** - работа с экспортом Excel

## ⚡️ Быстрый старт

### Локальная разработка

1. **Клонирую репозиторий**
   ```bash
   git clone <repository-url>
   cd rst_bot
   ```

2. **Создаю виртуальное окружение**
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/Mac
   # или
   venv\\Scripts\\activate  # Windows
   ```

3. **Устанавливаю зависимости**
   ```bash
   pip install -r requirements.txt
   ```

4. **Настраиваю переменные окружения**
   ```bash
   cp .env.example .env
   # Отредактирую .env с реальными значениями
   ```

5. **Оптионально: синхронизирую сотрудников из Excel**
   ```bash
   python sync_employees.py
   ```

6. **Запускаю бота**
   ```bash
   python bot.py
   ```

### Развертывание в Docker

```bash
docker-compose up --build
```

## ⚙️ Настройка

### Переменные окружения

Все обязательные переменные на основе `.env.example`:

- **TELEGRAM_TOKEN** - токен бота от @BotFather
- **TARGET_CHAT_ID** - ID канала для отправки заявок
- **FROM_EMAIL** - email адрес для отправки кодов
- **EMAIL_PASSWORD** - пароль от email (для Yandex – app password)

### Офис Excel синхронизация

Файл phonebook.xlsx должен содержать:
- Колонка '**ФИО**' - полное имя сотрудника
- Колонка '**E-mail**' - электронный адрес
