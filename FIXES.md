# Исправления проекта rst_bot

## ✅ Исправленные проблемы

### 1. **Синтаксические и логические ошибки**
- ❌ ~~Случайный символ "ы" в файле handlers.py~~
- ❌ ~~Опечатка в сообщении об ошибке (лишний символ ':')~~
- ✅ Исправлены

### 2. **Управление заданиями (job_queue)**
- **Проблема**: Использовался `context.job_queue.stop()`, что停止 всю очередь заданий
- **Решение**: Изменено на сохранение ссылки на задание и использование `job.schedule_removal()` для отмены конкретного задания
- **Файлы**: handlers.py (функции `start()` и `show_sub_problem_type_menu()`)

### 3. **Обновление зависимостей**
- **Проблема**: Использовалась устаревшая версия `python-telegram-bot==13.7`
- **Решение**: Обновлено на версию `20.7` с актуальной документацией и API
- **Файл**: requirements.txt
- **Другие обновления**:
  - `python-dotenv`: 0.19.0 → 1.0.0
  - `requests`: 2.26.0 → 2.31.0
  - `pandas`: 2.0.3 → 2.1.3

### 4. **Проверка переменных окружения**
- **Проблема**: Отсутствовали проверки на наличие обязательных переменных окружения
- **Решение**: Добавлены проверки в `config.py` с выбросом исключения `ValueError`, если переменные не установлены
- **Проверяемые переменные**:
  - TELEGRAM_TOKEN
  - TARGET_CHAT_ID
  - FROM_EMAIL
  - EMAIL_PASSWORD

### 5. **Пути к базам данных**
- **Проблема**: Жестко закодированные пути `/data/` работают только в Docker
- **Решение**: Добавлена логика в `config.py` для поддержки обеих сценариев:
  - **В Docker**: используется `/data/`
  - **При локальной разработке**: используется `.data/` в корне проекта
- **Файлы**: config.py, logging_config.py, sync_employees.py, database.py

### 6. **Индексы в базе данных**
- **Проблема**: Отсутствовали индексы для часто используемых полей
- **Решение**: Добавлены индексы:
  - **Таблица users**: индексы на `chat_id` и `phone`
  - **Таблица employees**: индексы на `full_name` и `email`
- **Файлы**: database.py, sync_employees.py

### 7. **Валидация номера телефона**
- **Проблема**: Отсутствовала валидация формата номера
- **Решение**: Добавлена функция `is_valid_phone()` в handlers.py с проверкой:
  - Начинается с '+7'
  - 12 символов в общей длине
  - Символы после '+7' - только цифры

### 8. **Сохранение пользователя в БД**
- **Проблема**: Пользователь не сохранялся в БД при вводе номера телефона
- **Решение**: Добавлен вызов `save_user_to_db()` в функции `get_phone()`
- **Файл**: handlers.py

### 9. **Функция get_phone()**
- **Проблема**: Функция импортировалась в bot.py, но не была определена
- **Решение**: Создана полная реализация функции `get_phone()` в handlers.py с:
  - Валидацией номера телефона
  - Сохранением в БД
  - Переходом к выбору категории проблемы

### 10. **Логирование**
- **Проблема**: Путь к логам был жестко закодирован
- **Решение**: Использование `LOG_DIR` из config.py
- **Файл**: logging_config.py

## 🚀 Использование

### Локальная разработка
```bash
# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # Linux/Mac
# или
venv\\Scripts\\activate  # Windows

# Установка зависимостей
pip install -r requirements.txt

# Создание файла .env на основе .env.example
cp .env.example .env
# Отредактировать .env с реальными значениями

# Запуск бота
python bot.py
```

### Docker
```bash
docker-compose up --build
```

## ⚠️ Важно

1. **Переменные окружения**: Обязательно установите все переменные из `.env.example`
2. **SMTP**: Для Yandex используйте app password, а не основной пароль
3. **Сотрудники**: Файл `phonebook.xlsx` должен содержать колонки 'ФИО' и 'E-mail'
4. **Логирование**: Логи хранятся в `bot.log` (в Docker - `/data/bot.log`)

## 📊 Структура БД

### Таблица users
```sql
CREATE TABLE users (
    id INTEGER PRIMARY KEY,
    chat_id BIGINT UNIQUE,
    name TEXT,
    phone TEXT UNIQUE
);
```

### Таблица employees
```sql
CREATE TABLE employees (
    id INTEGER PRIMARY KEY,
    full_name TEXT,
    email TEXT UNIQUE
);
```
