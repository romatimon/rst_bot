# Инструкция по тестированию Telegram бота

## Предварительные требования

1. **Python 3.7+** установлен
2. **Зависимости** установлены: `pip install -r requirements.txt`
3. **TELEGRAM_TOKEN** установлена как переменная окружения или в `.env`

## Способ 1: Запуск локально

```bash
# Перейти в директорию проекта
cd f:\dev\rst_bot

# Установить зависимости
pip install -r requirements.txt

# Запустить бот
python bot.py
```

Ожидаемый результат:
```
INFO:root:БОТ ЗАПУЩЕН И ГОТОВ К РАБОТЕ
```

## Способ 2: Запуск с Docker

```bash
# Собрать образ
docker build -t rst-bot .

# Запустить контейнер
docker run --rm \
  -e TELEGRAM_TOKEN="YOUR_TOKEN" \
  -e TARGET_CHAT_ID="YOUR_CHAT_ID" \
  -e FROM_EMAIL="your@email.com" \
  -e EMAIL_PASSWORD="your_password" \
  -v /data:/data \
  rst-bot
```

## Тестирование функциональности

### 1. Проверка запуска
- Бот должен начать polling и ожидать команд
- Нет ошибок в консоли

### 2. Команда /start
1. Отправить `/start` боту
2. Бот должен проверить существует ли пользователь
   - Если новый: запросить имя
   - Если существует: подтвердить номер телефона

### 3. Ввод имени сотрудника
1. Отправить полное имя сотрудника (например: "Иван Петров")
2. Бот должен:
   - Проверить наличие сотрудника в базе
   - Отправить код на email
   - Запросить ввод кода

### 4. Проверка кода
1. Посмотреть логи или консоль терминала для получения кода
2. Отправить 4-значный код
3. Бот должен подтвердить и перейти к запросу номера телефона

### 5. Ввод номера телефона
1. Отправить номер в формате +7 XXXXXXXXXX (12 цифр)
2. Бот должен:
   - Подтвердить номер
   - Сохранить пользователя в БД
   - Показать меню выбора проблемы

### 6. Выбор категории проблемы
1. Выбрать одну из категорий (кнопка):
   - Эл. почта 📧
   - ПО 'Синтез' 💙
   - Проблема с ПК 🖥️
   - Телефон ☎️
   - Принтеры, МФУ 🖨️
   - Удаленный доступ 🌐
   - Другое ❓

2. Бот должен перейти к выбору подкатегории

### 7. Выбор подкатегории
1. Выбрать подкатегорию из предложенных
2. В зависимости от выбора:
   - Для "Удаленного доступа": запросить номер AnyDesk
   - Для "Другое": запросить текстовое описание
   - Иначе: отправить данные в техподдержку

### 8. Завершение диалога
1. Должно отобразиться сообщение:
   ```
   Данные успешно отправлены в техническую поддержку. 
   В ближайшее время коллеги свяжутся с Вами для решения проблемы.
   ```
2. Диалог должен завершиться

## Проверка на ошибки

### Проверить отсутствие предупреждений о "coroutine was never awaited":

```bash
# Запустить с выводом всех предупреждений
python -W all bot.py
```

Должны отсутствовать сообщения вроде:
```
RuntimeWarning: coroutine 'Message.reply_text' was never awaited
```

### Проверить синтаксис файлов:

```bash
python -m py_compile bot.py handlers.py config.py database.py email_service.py logging_config.py send_support_chanel.py
```

Если ошибок нет - все хорошо.

## Отладка проблем

### Проблема: "ModuleNotFoundError: No module named 'telegram'"
**Решение:**
```bash
pip install python-telegram-bot==20.7
```

### Проблема: "TELEGRAM_TOKEN is not set"
**Решение:**
```bash
# Linux/Mac
export TELEGRAM_TOKEN="your_token"

# Windows (CMD)
set TELEGRAM_TOKEN=your_token

# Windows (PowerShell)
$env:TELEGRAM_TOKEN="your_token"
```

### Проблема: Ошибки с базой данных
**Решение:**
```bash
# Убедиться что директория существует
mkdir -p .data

# Удалить старую БД для пересоздания
rm -f .data/databases.db
# (или удалить файл вручную в Windows)
```

### Проблема: Ошибки SMTP при отправке кодов
**Решение:**
- Проверить учетные данные email (FROM_EMAIL, EMAIL_PASSWORD)
- Убедиться что SMTP сервер доступен
- Проверить логи для деталей ошибки
